## Сложности при реализации проекта

После настройки окружения агент реализовал весь проект (Sinatra-сервер, сервисы, виджеты, тесты).
В процессе вскрылся ряд нетривиальных проблем.

### Технические несовместимости версий

- **Rack 3.x убрал `Rack::Handler` и `Rack::Server`** — системные тесты падали с `uninitialized constant`.
  Пришлось переключиться на `Puma::Server.new` напрямую и в тестах, и в `app.rb`.
- **Ferrum: API отличается от документации** — `page.viewport_size = [w, h]` не существует,
  правильно `page.resize(width:, height:)`. Скриншот по умолчанию возвращает base64, а не бинарник —
  нужен явный `encoding: :binary`.
- **Playwright: версии ruby-client и CLI должны совпадать** — `playwright-ruby-client 1.58` требует
  `npx playwright@1.58.2`, иначе падает при подключении.
- **Sinatra 4.x host_authorization** — `disable :host_authorization` устанавливает `false`, но
  middleware ожидает Hash и падает с `TypeError`. Правильно: `set :host_authorization, { permitted_hosts: [] }`.

### Ошибки в порядке инициализации

- **`ClaudeTokensService`**: `super` вызывался до `@api_key = api_key`, поэтому `build_connection`
  получал `nil`. Исправление — сначала присвоить переменную, потом вызывать `super`.

### Потеря кода при автоисправлении

- **RuboCop удалил `def binding = super`** при авто-фиксе (считал лишним). Без него ERB-шаблон
  переставал работать. Восстановили как `def get_binding; binding; end` с обновлением вызова.

### Архитектурная недоработка

- **`app.rb` не запускал сервер** — файл был загрузчиком констант без стартового кода.
  `bundle exec ruby app.rb` завершался мгновенно без вывода. Исправили добавлением блока
  `if __FILE__ == $PROGRAM_NAME` с запуском через `Puma::Server`.

### Несовпадение ключей в DataStore

- Данные записывались с символьными ключами, читались через JSON-файл уже со строковыми.
  Исправление — нормализация через `JSON.parse(JSON.generate(value))` при записи.
