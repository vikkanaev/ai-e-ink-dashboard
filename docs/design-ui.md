# UI дизайн дашборда

## Ограничения e-ink дисплея

| Параметр | Значение |
|---------|---------|
| Разрешение | 800 × 480 px |
| Цвета | 1-bit (только чёрный и белый) |
| Дисплей | 7.5" e-Paper, контроллер UC8179 |
| Обновление | каждые 10 минут |

Следствия для дизайна:
- Никаких градиентов, теней, полупрозрачности
- Серый имитируется только диагональной штриховкой (dithering) — использовать минимально
- Чёткие контуры и крупный текст; мелкие детали теряются при 1-bit квантизации
- Нет анимаций; каждый пиксель — это навсегда до следующего обновления

## Виджет: Calendar

Отображает события текущего дня из Google Calendar.

```
┌────────────────────┐
│ СЕГОДНЯ            │  ← заголовок виджета (13px bold, uppercase)
├────────────────────┤
│                    │
│  10:00             │  ← время (14px bold)
│  Standup           │  ← название (14px normal)
│                    │
│  ──────────────    │  ← разделитель между событиями (1px, 80% ширины)
│                    │
│  14:00             │
│  1:1 с Артёмом     │
│                    │
│  ──────────────    │
│                    │
│  17:00             │
│  Deploy prod       │
│                    │
│  ──────────────    │
│                    │
│  (нет событий)     │  ← если список пуст
│                    │
└────────────────────┘
```

- Максимум **6 событий** в списке; остальные обрезаются
- Если название не влезает в одну строку — обрезается с `…` (ellipsis)
- Текущее событие (идёт прямо сейчас) — время выделяется **инверсией** (белый текст на чёрном фоне)
- Если событий нет — по центру мелким текстом: `нет событий`

## Виджет: WakaTime

Отображает продуктивность за сегодня.

```
┌──────────────────────────────────────┐
│ WAKATIME                             │  ← заголовок
├──────────────────────────────────────┤
│                                      │
│         4ч 23м                       │  ← итого (36px bold, по центру)
│       за сегодня                     │  ← подпись (13px, по центру)
│                                      │
│  ████████████████░░░░░░    73%       │  ← прогресс-бар + процент от цели
│                                      │
│  ────────────────────────────────    │
│                                      │
│  Ruby    ██████████░░░░░░   2ч 10м  │
│  JS      ████████░░░░░░░░   1ч 05м  │
│  Python  █████░░░░░░░░░░░     45м   │
│  YAML    ██░░░░░░░░░░░░░░     18м   │
│                                      │
└──────────────────────────────────────┘
```

**Прогресс-бар (главный):**
- Полная ширина: `360 - 24 = 336 px`
- Высота: **12 px**
- Заполнение: пропорционально `total / daily_goal` (goal = 6 часов по умолчанию)
- Символ заполнения: сплошной прямоугольник `█`, фон — пустой прямоугольник `░`
- В CSS: `background: #000; height: 12px` + пустой фон `border: 1px solid #000`

**Мини-бары языков:**
- Ширина бара: **160 px** (фиксированная), пропорционально максимальному языку
- Высота: **8 px**
- Показывать максимум **4 языка**
- Название: 60 px, бар: 160 px, время: оставшееся место

## Правила 1-bit рендеринга

Изображение генерируется через HTML/CSS → Ferrum screenshot → 1-bit квантизация.

1. **CSS цвета:** только `#000000` и `#ffffff`. Никаких `#333`, `rgba`, `opacity`.
2. **Шрифты:** без сглаживания — `-webkit-font-smoothing: none` в CSS.
3. **Границы:** минимум `1px solid #000`; тонкие линии на 1-bit могут пропасть.
4. **Прогресс-бары:** реализовать как `<div>` с фиксированной высотой и `background: #000`, без `border-radius`.
5. **Порог квантизации:** 128 (пиксель >= 128 → белый, < 128 → чёрный). Настраивается в `ImageRenderer::THRESHOLD`.

## Состояния ошибок

Если данные от сервиса недоступны (DataStore вернул nil):

```
┌────────────────────┐
│ WAKATIME           │
├────────────────────┤
│                    │
│   нет данных       │  ← по центру виджета, 14px
│                    │
└────────────────────┘
```

Текст `нет данных` — по вертикальному и горизонтальному центру области виджета.
